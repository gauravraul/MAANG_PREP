# System Design Interview Q&A – Set 8 (Extreme / Final Boss Level)

---

## Q1: Design a Global AI Inference Platform

### Answer:

### Requirements:
- Deploy multiple ML models
- Real-time inference (<100ms)
- Model versioning
- A/B testing
- Multi-tenant isolation

### Architecture:
Client → API Gateway → Model Router → Inference Runtime → GPU/TPU Pool  
Inference Runtime → Feature Store → Cache

### Scheduling:
- Route by model_name + version
- Load-based GPU assignment
- Pre-warm models

### Scaling:
- Autoscale based on QPS
- Horizontal pod scaling
- Model-specific queues

---

## Q2: Design a Global Streaming Analytics Engine (Flink-like Backbone)

### Answer:

### Requirements:
- Process millions of events/sec
- Stateful operators
- Exactly-once semantics

### Architecture:
Event Sources → Ingest → Stream Processor → Sinks

### Concepts:
- Checkpointing
- Event time/process time
- Windowing

### Fault Tolerance:
- Periodic state snapshots
- Replay from offsets
- Deterministic operators

---

## Q3: Design a Internet-Scale DNS System

### Answer:

### Requirements:
- Resolve queries globally
- Handle DDOS
- Fast propagation

### Architecture:
Root Servers → TLD Servers → Authoritative Servers → Caches

### Optimization:
- Anycast
- Aggressive caching
- Geo-routing

---

## Q4: Design a Zero-Trust Security Architecture

### Answer:

### Requirements:
- Authenticate every request
- Continuous verification
- Minimal trust

### Architecture:
Identity Provider → Policy Engine → Enforcement Points

### Techniques:
- mTLS
- Device posture checks
- Least privilege

### Choices:
