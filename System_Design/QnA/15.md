# System Design Interview Q&A – Set 7 (Distinguished Engineer / Bonus Set)

---

## Q1: Design a Planet-Scale Edge Compute Platform (Like Cloudflare Workers)

### Answer:

### Requirements:
- Run user code close to users
- Millisecond cold starts
- Multi-tenant isolation
- Secure execution

### Architecture:
Client → Anycast Network → Edge Runtime → Global Control Plane

### Runtime:
- WASM-based execution
- Memory & CPU sandboxing
- Per-request isolation

### Scheduling:
- Nearest PoP selection
- Load-aware routing

### State:
- Stateless by default
- Global KV with eventual consistency
- Durable objects for strong consistency

---

## Q2: Design a Planet-Scale Object CDN (Beyond Traditional CDN)

### Answer:

### Requirements:
- Read-heavy workloads
- Origin shield
- Strong cache guarantees

### Architecture:
Edges → Regional Clusters → Core Datacenters

### Features:
- Content-aware routing
- Predictive prefetch
- Tiered eviction policy

---

## Q3: Design a Federated Identity System (Cross-Company SSO)

### Answer:

### Requirements:
- One login for many orgs
- Secure delegation
- Token trust

### Flow:
Org A Login → Token Authority → Org B Resource Access

### Standards:
- OAuth2
- OpenID Connect
- SAML

# Distinguished Engineer Answer Model

When answering these:

- Think in layers  
- Consider failure as default  
- Control blast radius  
- Optimize globally, not locally  
- Assume partial failure always  
- Reason with real numbers
